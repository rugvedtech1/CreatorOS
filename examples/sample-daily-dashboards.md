# Sample Daily Dashboards

Worked examples showing how the engines combine in practice, following
the flow defined in `SKILL.md` and `engines/daily-intelligence.md`. Use
these as a reference for output shape and depth — not as content to copy
verbatim for a real creator.

---

## Example 1 — Photography creator, rainy day, Kolhapur

**Profile:** Photography, beginner-intermediate, 4K followers, posts
3x/week, shoots on phone + entry DSLR, goal: grow followers + brand.

**Today's context:** Heavy rain forecast for the afternoon, clearing by
early evening (source: creator-provided, no live tool access this
session).

**Dashboard:**

1. **Reel idea 1** — "Shooting reflections after the rain" — hook:
   "Everyone puts their camera away when it rains. That's the mistake."
   Body: 3 specific reflection-shot techniques (puddle-level angle,
   post-rain golden hour timing, using wet surfaces for color bounce).
   CTA: "Follow for more shots most people walk past."
2. **Reel idea 2** — Gear-agnostic tip: "You don't need a DSLR for this
   shot" — phone-only reflection technique, positions the creator for
   the beginner segment of their audience specifically.
3. **Carousel idea** — "5 settings to change before a rainy shoot" —
   5 slides, one setting each, with the specific value and why.
4. **Story ideas** — Morning: poll ("shooting today or waiting it out?");
   Afternoon: BTS of gear prep for wet conditions; Evening: single
   best shot with a caption on what almost went wrong.
5. **Suggested posting time** — early evening, aligned with when the rain
   clears and audience is home scrolling post-work.
6. **Hashtags** — mix of location-specific (#KolhapurPhotography) and
   technique-specific (#reflectionphotography), avoiding purely generic
   high-volume tags.

## Example 2 — Tech creator, AI model release day

**Profile:** Tech/AI niche, intermediate, 12K followers, posts daily,
goal: authority + sponsorships.

**Today's context:** A major AI lab released a new model this morning
(source: live search, verified before use per `engines/trend-research.md`).

**Dashboard:**

1. **Reel idea 1** — Breaking news reel, hook: "This dropped 2 hours ago
   and nobody's talking about the actual limitation yet." Body: hands-on
   first impression including one real limitation found, not pure hype.
   CTA: "Follow — I'll post the full breakdown once I've tested it more."
2. **Reel idea 2** — Comparison against the leading incumbent model on
   one specific, concrete task (not a generic benchmark claim).
3. **Carousel idea** — "3 things this changes for [specific audience
   segment, e.g. indie developers]" — practical implications, not
   speculation framed as fact.
4. **Story ideas** — Immediate: quick reaction poll ("hyped or
   skeptical?"); Midday: screen-recorded first test; Evening: question
   box for "what should I test next."
5. **Suggested posting time** — as close to real-time as possible; speed
   matters more than polish for breaking tech news per
   `niches/tech.md`.
6. **Hashtags** — the model's specific name/tag plus the creator's
   established niche tags, avoiding tag-stuffing.

## Example 3 — Food creator, local festival week

**Profile:** Food/street food niche, professional, 40K followers, posts
5x/week, goal: sponsorships + community.

**Today's context:** A local food festival is running this week
(source: creator-provided).

**Dashboard:**

1. **Reel idea 1** — "Hidden stalls locals actually go to" —
   deliberately avoids the obvious main-entrance vendors covered by
   every other creator at the festival, per `niches/food.md` originality
   guidance.
2. **Reel idea 2** — First-bite reaction series across 3 stalls, cut
   together — genuine reaction prioritized over scripted praise.
3. **Carousel idea** — "Festival map: what to eat in what order" —
   practical, save-worthy for someone planning their visit.
4. **Story ideas** — Live coverage BTS throughout the day; evening
   question box ("which stall should I revisit tomorrow?").
5. **Suggested posting time** — same-day/near-live for the Reels to
   capture festival urgency; Carousel can post the following morning for
   planning value.
6. **Hashtags** — official festival tag (if one exists) plus
   city+food-specific tags.

## What these examples demonstrate

- Every dashboard starts from **real, sourced context** — never invented
  trends or fabricated urgency (`ethics/safety-guardrails.md`)
- Hooks and angles are **specific to the creator's actual niche and
  audience segment**, not generic templates
- Each dashboard explains **why** a choice was made (timing, format,
  angle) — the creator should learn the reasoning, not just receive
  output
- Format variety (Reel/Carousel/Story) is used deliberately per what
  each format is actually good at, not applied uniformly
