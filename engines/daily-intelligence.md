# Daily Intelligence Engine

Use this file when the creator asks "what should I post today," wants a
daily/weekly content plan, or wants ideas tied to current conditions
(weather, season, trends, local events).

## Step 1 — Establish real context first, never invent it

This is the most important rule in this file: do not fabricate "trending"
topics, weather, or events. Fabricated context produces content that looks
current but isn't, which damages trust the moment a follower notices.

- If you have live tool access (web search, etc.) in this environment,
  use it to check: current weather for the creator's location, any local
  festivals/events this week, and what's trending in their specific niche
  right now.
- If you do not have tool access, or the search comes back thin, ask the
  creator directly: "What's the weather/vibe there today?" or "Anything
  trending in your space this week that I should know about?" Treat their
  answer as the source of truth.
- Never present an invented trend as if it were verified real-time data.

## Step 2 — Combine context with niche

Once you have real context + the creator's niche (from their profile),
cross-reference using this pattern:
[Context signal] + [Niche] → [Content angle]

Examples of the pattern (illustrative, not a lookup table to copy verbatim
— generate fresh angles for the creator's actual situation):

- Heavy rain + Photographer → reflection shots, golden hour right after
  rain clears, indoor moody lighting alternative if outdoor isn't viable
- Major AI model release + Tech creator → first-impressions reel, "what
  this means for [specific audience]" breakdown, hands-on comparison
- Local food festival + Food creator → street-food tour, "hidden stalls
  locals actually go to" angle (more save-worthy than generic coverage)
- Cold/winter onset + Fitness creator → indoor routine pivot, "why your
  motivation drops in winter" (psychology angle, more shareable than a
  plain workout post)

The goal is always specificity over genericness — "photography tips" is
forgettable, "how to shoot after rain in [their city] without a filter"
is useful and save-worthy.

## Step 3 — Build the dashboard output

Structure every daily dashboard as:

1. **Today's context** — the real weather/trend/event info used, and its
   source (search result vs. creator-provided)
2. **1-2 Reel ideas** — angle + one-line hook (full script only if asked,
   via `engines/script-writing.md`)
3. **1 Carousel idea** — angle + suggested slide count and structure
4. **1-3 Story ideas** — quick, low-effort formats from
   `engines/story-engine.md` (poll, BTS, countdown, etc.)
5. **Suggested posting time** — based on their stated audience/timezone,
   not a generic "9am and 7pm" unless that's genuinely well-suited
6. **Relevant hashtags** — a mix of niche-specific and broad, not just
   high-volume tags that bury the post in oversaturated competition

## Step 4 — Weekly/monthly extension

If asked for a weekly or 30-day plan instead of a single day: don't repeat
the same content angle daily. Vary format (Reel/Carousel/Story mix) and
vary content pillar (education, entertainment, behind-the-scenes,
community, promotional) across the week so the feed doesn't feel
repetitive. A healthy default mix leans heavily educational/entertaining
with promotional content kept to a small minority — ask the creator's
actual goal mix rather than assuming.

## Guardrail

Every idea generated here must still pass `ethics/safety-guardrails.md`
before being presented — e.g., never suggest riding a trend through
misleading framing or engagement-bait tactics disguised as "trend-jacking."
