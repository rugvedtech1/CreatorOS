# Script Writing Engine

Use this file whenever the creator wants a Reel script, video script, or
any content that needs to hold attention from first frame to last. This
applies whether they ask directly ("write me a script") or when the daily
dashboard needs a script attached to an idea.

## The five-part structure

Every script follows this sequence. Not every part needs multiple sentences —
on Reels the whole thing may be 15-30 seconds of spoken/on-screen content.

1. **Hook (0-2 seconds)**
   The single most important part. Must stop the scroll before the viewer
   consciously decides to keep scrolling. Effective hook types:
   - Bold claim/contradiction ("Everyone told you X. They're wrong.")
   - Immediate visual payoff (show the result before explaining it)
   - Direct question that implicates the viewer ("Still doing X wrong?")
   - Open loop ("The mistake in this photo cost me my biggest client.")
   Avoid: slow intros, "hey guys welcome back," any setup before value.

2. **Pattern Interrupt (2-4 seconds)**
   Reinforces the hook so the viewer commits to watching. Often a visual
   cut, a text overlay restating the stakes, or a tone shift. This is what
   separates a hook that gets a half-second glance from one that earns
   3 more seconds of attention.

3. **Body (the value)**
   Deliver on the hook's promise. One clear idea per script — resist
   cramming in three tips when one well-explained tip retains better.
   Use concrete, specific detail (numbers, named steps, before/after)
   over vague advice. Specificity is what makes content feel worth saving.

4. **CTA (Call to Action)**
   One CTA, not three. Match the CTA to the goal:
   - Growth goal → "Follow for [specific recurring value], not just Part 2"
   - Community goal → ask a real question, not "comment below!"
   - Save-worthy content → tell them explicitly why to save it
   - Sponsorship/sales goal → link in bio / soft mention, not hard sell

5. **Follow-up Content Ideas**
   Always suggest 1-2 natural follow-up scripts. Series thinking builds
   appointment viewing and follow-through — a good script suggests its
   own sequel.

## Optimization targets — ask which one matters most for this piece

Different scripts should optimize for different signals, since chasing all
of them at once dilutes the script:

- **Watch time / completion**: shorter is often better than longer;
  reward the viewer for staying (payoff must land, not fizzle)
- **Shares**: needs an emotional or identity-relevant core ("this is SO me"
  or "my friend needs to see this")
- **Saves**: needs to feel like reference material — tutorials, lists,
  frameworks people will want to revisit
- **Comments/community**: needs genuine ambiguity or a real question,
  not a rhetorical one

Ask the creator (or infer from their stated goal) which of these matters
most before writing, since it changes pacing and CTA choice.

## Output format when writing a script

Always deliver:
1. The script itself, broken into the 5 labeled parts above
2. On-screen text suggestions (separate from spoken words, if applicable)
3. Suggested shot/visual for each part (even a rough description helps)
4. Why the hook works (the psychological principle behind it) — the
   creator should learn the pattern, not just receive the output

## Common failure modes to avoid

- Hook that's clever but doesn't match the body's actual content
  (causes drop-off and hurts completion rate)
- Multiple CTAs competing with each other
- Body that explains *around* the value instead of directly stating it
- Generic hooks that could apply to any creator in any niche — always
  make the hook specific to this creator's actual niche and voice
